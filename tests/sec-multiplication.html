Using the two pictures below, explain why 2×3=3×2 .

Solution for Problem 7:

The picture on the left has 2 rows. Each row has 3 squares. So the total number of squares is 3+3 , which is 2×3 .

The picture on the right has 3 rows. Each row has 2 squares. So the total number of squares is 2+2+2 , which is 3×2 .

But the picture on the right is just a rotation of the picture on the left. Turning a picture doesn’t change how many squares are in it. So we conclude that 2×3=3×2 .

Our example suggests that we can reverse the numbers in multiplication just as we can in addition. In other words, multiplication is commutative. As with addition, this is not a proof that commutativity works for all numbers, but our example does give us a good idea why we have a×b=b×a for all positive numbers a and b. We call this the commutative property of multiplication.

We will often write a×b as a⋅b , using a centered dot. One reason is that it is quicker to write a dot than a cross. Another reason is that the cross × looks too much like the letter x, which is used a lot in algebra. So we can write a⋅b=b⋅a for all numbers a and b.

We will often go a step further, writing a⋅b as ab. This notation is even quicker to write. We can’t always leave out the dot, though. For example, to express 3⋅4 , we can’t write 34, because that number is thirty-four. In such cases, we will write either 3⋅4 or 3(4) for multiplication.

So we can write the commutative property of multiplication as:

Multiplication is commutative: Let a and b be numbers. Then
ab=ba.

Is multiplication associative too? The next example is slightly trickier:

By counting the dots in the picture below in two different ways, explain why

(2×3)×4=2×(3×4).
Solution for Problem 8:

On one hand, there are (2×3) squares. Each square has 4 dots. So the total number of dots is (2×3)×4 .

On the other hand, there are 2 rows of squares. Each row has (3×4) dots. So the total number of dots is 2×(3×4) .

We have counted the same dots in two different ways. So we conclude that

(2×3)×4=2×(3×4).

Problem 8 is an example of the associative property of multiplication:

Multiplication is associative: Let a, b, and c be numbers. Then
(ab)c=a(bc).

Together, the commutative and associative properties let us multiply numbers in any order, just as we can add numbers in any order. Even with more than 3 numbers, this any-order principle applies. Let’s see this in action.

Compute 25×125×4×6×8 .

Solution for Problem 9:

Computing 25×125 takes work, so let’s reorder the numbers to simplify our task. Let’s look for easy pairs such as 4×25 that lead to nice round numbers. For example, let’s try

6×(4×25)×(8×125).

The product of 4 and 25 is 100. The product of 8 and 125 is 1000. So we are left with

6×100×1000.

The answer is 600,000 .

We’ve already seen that zero is a special number for addition: adding zero does nothing. Is there a similar number for multiplication?

Using the picture below, explain why 2×1=2 .

Solution for Problem 10:

On one hand, there are 2 rows. Each row has 1 square. So there are 2×1 squares in all. On the other hand, there are 2 squares. So we conclude that 2×1=2 .

This example suggests that multiplying a number by 1 does not change the number, and this leads to another rule:

Multiplying by 1: Let a be a number. Then
1a=a.

Before we continue with properties of arithmetic, we need to understand how to combine different operations. We know that if we are just adding or just multiplying, we can move the numbers around and we can perform the operations in any order. But it’s a little more complicated if we have both types of operation in the same calculation. For instance, what does 5+6×7 mean?

  1. Compute (5+6)×7 .

  2. Compute 5+(6×7) .

Solution for Problem 11:

  1. The parentheses tell us to add first:

    (5+6)×7=11×7=77.

  2. This time, the parentheses tell us to multiply first:

    5+(6×7)=5+42=47.

We see that (5+6)×7̸ =5+(6×7) . (The symbol “≠ ” means “not equal to.”)

So what if we were presented with just the expression 5+6×7 , without any parentheses? We need rules for what order to perform different operations. These rules are shown below. (We’ve only covered addition and multiplication so far; we will discuss the other operations later.)

Order of operations: Perform the operations in an expression in the following order.

  1. Evaluate expressions inside parentheses first.

  2. Compute powers. (We cover powers in Chapter 2.)

  3. Multiply and divide from left to right.

  4. Add and subtract from left to right.

So, to compute 5+6×7 , we multiply first, then add:

5+6×7=5+42=47.

In other words, the calculation looks like part (b) of Problem 11.

Next, we turn to a powerful rule that connects addition and multiplication. As usual, we’ll motivate the rule with a simple example.

Using the picture below, explain why 2×(3+4)=(2×3)+(2×4) .

Solution for Problem 12:

On one hand, there are 2 rows. Each row has (3+4) squares. So altogether there are 2×(3+4) squares.

On the other hand, there are (2×3) dark squares, and there are (2×4) light squares. So altogether there are (2×3)+(2×4) squares.

We have counted the same squares in two different ways. So we conclude that

2×(3+4)=(2×3)+(2×4).

Problem 12 is an example of a very useful rule that relates multiplication and addition. For any three numbers, a, b, and c we have

a×(b+c)=(a×b)+(a×c).

The multiplication is distributed (handed out) to the two parts of the addition. For that reason, this property is called the distributive property of multiplication over addition, or the distributive property for short.

Because multiplication is commutative, we can reverse each of the products above, and write

(b+c)×a=b×a+c×a.

In other words, the distributive property works when the sum is first, (b+c)×a , or last, a×(b+c) .

To reduce clutter, we can write the distributive property much more simply:

Multiplication distributes over addition: Let a, b, and c be numbers. Then
a(b+c)=ab+acand(b+c)a=ba+ca.

The equations a(b+c)=ab+ac and (b+c)a=ba+ca are really the same equation. We include them both to highlight the fact that the distributive property can be used when the sum is first or last.

Let’s explore the distributive property further. What can this property do for us?

Compute 51⋅9+51⋅31 .

Solution for Problem 13:

Instead of separately computing 51⋅9 and 51⋅31 , we can use the distributive property:

51⋅9+51⋅31=51(9+31).

On the right side, the sum 9+31 is 40, so our answer is equal to 51⋅40 .

Instead of computing 51⋅40 , we can use the distributive property again to make the multiplication simpler. Writing 51=50+1 lets us use the distributive property as

51⋅40=(50+1)⋅40=50⋅40+1⋅40.

The advantage of using the distributive property as we just did is that the products 50⋅40 and 1⋅40 are both easy to compute! Our answer is

50⋅40+1⋅40=2000+40=2040.

As we saw in Problem 13, we can use the distributive property “in either direction.” Using the distributive property to rewrite a(b+c) as ab+ac is called expanding. Using the distributive property to rewrite ab+ac as a(b+c) is called factoring.

For example, in the first step of the solution to Problem 13, we factored 51⋅9+51⋅31 to write it as the simpler 51(9+31) . In the second step, we expanded (50+1)⋅40 to write it as the easier-to-compute 50⋅40+1⋅40 .

Factoring means using the distributive property to rewrite something of the form ab+ac as something of the form a(b+c) . For example,
51⋅9+51⋅31=51(9+31).
In this example, the number 51 is called a common factor of both products on the left. We say that we “factor out 51” when we write 51⋅9+51⋅31 as 51(9+31) . We might also say that we “pull out a common factor of 51.”

So far, we have used the distributive property with sums of two numbers. What about longer sums?

What is the value of 17⋅13+51⋅13+32⋅13 ?

Solution for Problem 14:

We know that we can use the distributive property if we have two products with a common factor (like ab and ac, which have the common factor a). So we can factor out 13 from the products 17⋅13 and 51⋅13 :

17⋅13+51⋅13+32⋅13=(17+51)⋅13+32⋅13.

But now the products (17+51)⋅13 and 32⋅13 also have a common factor of 13. So we can use the distributive property again:

(17+51)⋅13+32⋅13=((17+51)+32)⋅13.

By the associative property of addition, we don’t need the inner set of parentheses, so our quantity is just (17+51+32)⋅13 . But 17+51+32 equals 100, so our answer is 100⋅13=1300 .

What Problem 14 shows us is that the distributive property works on sums of three (or more) numbers too! That is, we can pull out a common factor from a sum of any number of products. Now that we’ve seen that this works, we don’t need to show all the steps like we did in the solution to Problem 14. Here’s how we can solve the problem more quickly:

Solution for Problem 14:

We see that all three numbers that we are summing (17⋅13 , 51⋅13 , and 32⋅13 ) have the common factor 13, so we can factor the sum:

17⋅13+51⋅13+32⋅13=(17+51+32)⋅13.

Since 17+51+32 equals 100, our answer is 100⋅13=1300 .

The distributive property works for a sum of any number of products. For example, if a, b, c, d, and e are any numbers, then:
ab+ac+ad+ae=a(b+c+d+e).
Similarly, we have
ba+ca+da+ea=(b+c+d+e)a.

You may have learned that multiplying by an integer is the same as “repeated addition.” For example,

4⋅7=7+7+7+7.

The distributive property, combined with the rule that 1×a=a for any number a, tells us why this is true. Since 4=1+1+1+1 , we have

4⋅7=(1+1+1+1)⋅7=1×7+1×7+1×7+1×7=7+7+7+7.

What happens when we multiply a number by zero? You probably already know what happens, but let’s try to see why it makes sense in the next problem.

Consider the multiplication facts below:

5×74×73×72×71×70×7=35=28=21=14=7=9999⎯⎯⎯⎯⎯⎯⎯⎯.

  1. Find a pattern in the multiplication answers.

  2. Assuming that the pattern continues, predict the answer to the last multiplication.

Solution for Problem 15:

  1. After the first equation, each number on the right-hand side of an equation is 7 less than the number above it.

  2. The last number that we see on the right-hand side is 7. Going down by another 7 brings us to 0. So we predict that 0×7 is 0.

    The idea that “multiplication by an integer is repeated addition” also suggests that 0×7 is 0. Just like 2×7 equals the sum of two 7’s, and 1×7 equals the sum of one 7, the product 0×7 equals the sum of no 7’s at all. Adding nothing gives us nothing, so again we expect that 0×7 is 0. We might also note that 0×7=7×0 , and the sum of seven 0’s is 0.

The same idea shows that multiplying any number by 0 results in 0. It doesn’t matter whether the starting number is small or large, a fraction or an integer, positive, negative, or zero—multiplying a number by 0 always results in 0. Multiplying by 0 “destroys” every number.

Multiplying by zero: Let x be a number. Then
0x=0.
